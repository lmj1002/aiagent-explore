# /review — 代码审查与合并决策

对当前分支的改动进行全面审查，输出审查报告和合并决策。

---

## 用法

```
/review                              # 审查当前分支的所有未合并改动
/review --files <path1> <path2>      # 只审查指定文件
/review --issue <issue-number>       # 审查指定 Issue 关联的分支
/review --pr <pr-number>             # 审查指定 PR
```

---

## 执行流程

### Step 1：获取 Diff

```bash
# 当前分支 vs 主分支的差异
git diff main...HEAD --stat
git diff main...HEAD
```

### Step 2：多维度审查

调度 `reviewer` 智能体，从以下维度逐一检查：

| 维度 | 检查内容 |
|------|----------|
| **正确性** | 逻辑错误、边界条件、空值处理、并发问题 |
| **性能** | N+1 查询、内存泄漏、不必要的循环、缓存缺失 |
| **安全** | SQL 注入、XSS、敏感信息泄露、权限校验 |
| **架构** | 职责单一、模块耦合、抽象层次、代码重复 |
| **可维护性** | 命名规范、注释质量、错误处理、日志 |

### Step 3：输出审查报告

---

---

## 禁止

- 不要只列出问题不给修复建议
- 不要跳过安全维度（即使是小改动）
- 不要在 REJECT 时给出模糊理由
