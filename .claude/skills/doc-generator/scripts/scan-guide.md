# 目录扫描与分析操作指南

## 步骤 1：扫描目录结构

### 工具调用

```
Glob(pattern="**/*", path="目标目录")   → 获取所有文件清单
Bash(ls -la "目标目录")                  → 查看隐藏文件（.env 等）及文件大小
```

### 补充检查

- 检查同级目录是否有共享资源（`.env`、公共模块）
- 检查上层目录是否有依赖的其他 agent 代码
- 用 `Glob` 扫描整个项目范围确认无遗漏

## 步骤 2：文件读取优先级

按以下顺序逐个 Read，每个文件读完后记录关键信息：

| 优先级 | 文件类型 | 读取重点 |
|--------|---------|----------|
| **P0** | 主程序 `.py` | 导入、工具定义、Agent 创建、执行入口 |
| **P1** | 配置 `.yaml` `.json` | 模型配置、工具声明、系统提示 |
| **P2** | 提示词/记忆 `AGENTS.md` `SKILL.md` | Agent 行为规范、写作模板 |
| **P3** | 辅助文件 `demo*.py` | 演示模式、额外配置示例 |
| **P4** | 输出/数据文件 `.csv` `.png` | 了解数据格式（确认即可，不用详读） |

## 步骤 3：组件识别清单

读代码时逐项识别以下模式，每识别到一个就记录下来：

### Agent 相关

- [ ] Agent 创建方式：`create_deep_agent` / `create_agent` / 自定义 Graph
- [ ] 调用参数：`model` / `tools` / `system_prompt` / `checkpointer` / `subagents`
- [ ] 后端类型：`FilesystemBackend` / `InMemorySaver` / 无后端

### 工具相关

- [ ] 每个 `@tool` 装饰的函数：名称、输入、输出、调用的后端 API
- [ ] 工具分配：哪些工具给主代理，哪些给子代理
- [ ] 特殊模式：Mock 模拟、双模式切换（直连/代理）

### 模型相关

- [ ] 主代理模型：本地（Ollama）还是云端（Gemini/OpenAI）
- [ ] 子代理模型：是否独立指定
- [ ] 加载方式：`load_llm()` 函数 / `init_chat_model` 字符串 / 硬编码
- [ ] 环境变量依赖：需要哪些 `*_API_KEY`

### 配置相关

- [ ] `load_dotenv` 路径（确认 `.env` 实际位置）
- [ ] 声明式配置（YAML）与代码式定义的映射关系
- [ ] `memory` / `skills` 参数引用的文件路径

## 步骤 4：绘制架构图

### 4.1 确定层级

```
层级 0: 用户输入
层级 1: 主代理（列出 model / tools）
层级 2: 工具 → 后端（每个工具一行）
层级 3: 子代理 → 子工具 → 后端（如有）
```

### 4.2 绘制规则

- 从上到下表示调用方向
- 用 `│` 连接上下级，`▼` 表示方向
- `task()` 委派标注在线上
- 框宽 ≈ 50 字符，内文居中

### 4.3 验证

- 每条线都能在代码中找到对应的调用关系
- 每个框内的信息与 import / 参数一致

## 步骤 5：追踪执行流程

从 `if __name__ == "__main__"` 开始，按以下思路追踪：

1. 初始化阶段：`load_dotenv()` → `load_llm()` → `create_agent()`
2. 任务输入：`HumanMessage(content=...)`
3. 执行阶段：agent 调用了哪些工具 / 子代理，顺序如何
4. 输出阶段：结果写入哪里（文件 / print / 图表）

用 ASCII 时序框表达，每步一个框，框内写具体操作。
