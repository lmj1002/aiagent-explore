# 格式规范

## 章节编号

- **一级章节**：使用中文数字 + 顿号（`一、二、三...`）
- **二级章节**：使用 `数字.数字` 格式（`1.1、1.2、4.3...`）
- **三级章节**：使用 `数字.数字.数字` 格式（`5.1.1...`），尽量不超过三级
- 章节间用 `---` 分隔

## 代码块

- 使用三个反引号围栏
- Python 代码标注 `python`，bash 命令标注 `bash`
- 配置/环境变量标注 `bash` 或 `ini`
- 工具描述中的 docstring（英文）保留原文，正文用中文说明

## 表格

- 用于对比、参数说明、依赖列表
- 第一行为表头，对齐格式：`|---|------|`
- 表格内容中文，字段名/代码用行内代码

示例：
```markdown
| 工具 | 行数 | 功能 | 后端 |
|------|------|------|------|
| `web_search` | 33-63 | DuckDuckGo 搜索 + 网页全文抓取 | 免费 |
```

## 行内代码

- 文件名：`` `agent.py` ``
- 函数名：`` `load_llm()` ``
- 参数名：`` `model` ``、`` `checkpointer` ``
- 模块名：`` `langchain_google_genai` ``
- 路径：`` `deep-research-agent/` ``

## ASCII 框图

### 字符集

```
┌ ┐ └ ┘ ├ ┤ ┬ ┴ ┼ │ ─ ▼ ▶
```

### 架构图规范

```
组件名（居中放在框内）
   │  (竖线表示调用/数据流)
   ▼  (箭头表示方向)
子组件或服务
```

- 框内文字居中对齐
- 用 `│` 连接上下的组件
- 用 `▼` 或 `▶` 表示方向，旁边标注方法名（如 `task()`）
- 同一层级的组件左右对齐
- 框宽度保持一致（通常 40-50 字符）

### 执行流程规范

```
┌─ 步骤N: 名称 ────────────────┐
│  具体操作描述                  │
│  可多行                        │
└──────────────┬────────────────┘
               ▼
┌─ 步骤N+1: 名称 ──────────────┐
│  子流程可缩进表示嵌套           │
│  ┌─ 子步骤 ───────────────┐   │
│  │  子操作描述              │   │
│  └─────────────────────────┘   │
└────────────────────────────────┘
```

## 中英文混排规则

- 中文正文中出现的英文专有名词保留原文（`Markdown`、`DuckDuckGo`）
- 代码注释保留原文，后面可附中文说明
- 工具名、类名、函数名不翻译
- prompt 内容如已在代码中被翻译为中文，文档中可用中文引用

## 排版

- 每个章节独立，之间有明确的分隔
- 关键概念用 `**粗体**` 强调
- 列表项用 `-` 开头
- 引用/注意事项用 `>` 块引用
