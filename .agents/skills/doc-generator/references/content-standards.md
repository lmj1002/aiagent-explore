# 内容标准

## 8 模块文档结构

### 一、项目概述

- **一句话描述**：这个项目是做什么的
- **核心能力**：3-5 条 bullet point，每条约 1 行
- **字数**：控制在一屏内（~10 行）

### 二、文件结构

- 用 ` ``` ` 代码块展示目录树
- 每个文件附 1 句话说明
- 标注哪些是可执行文件、哪些是配置文件

### 三、架构总览

- 用 ASCII 框图展示组件关系
- 必须包含：主代理 → 工具 → 后端服务 的完整链路
- 如有子代理，用分层结构表达

### 四、核心组件详解

逐个深入分析每个核心组件：

| 每个组件必须包含 | 说明 |
|-----------------|------|
| **位置** | 文件名:行号范围 |
| **功能** | 1-2 句中文描述 |
| **代码片段** | 关键的 5-15 行代码（非完整函数） |
| **技术点** | 表格列出 2-5 个值得注意的技术点 |

组件类型优先级：
1. 工具函数（`@tool` 装饰器）
2. Agent 创建函数
3. 模型加载函数
4. 配置加载逻辑
5. 技能文件（SKILL.md）
6. 声明式配置文件（YAML）

### 五、完整执行流程

- 从启动入口到最终输出的完整路径
- 每个步骤用 ASCII 框表示
- 步骤名用动词开头（初始化、接收任务、委派研究...）
- 子步骤用缩进表达嵌套

### 六、关键技术点

- 从代码中提炼 5-10 个独立的技术点
- 每个技术点配表格说明：名称 / 说明
- **不是**罗列代码行，而是解释为什么这么写的设计意图
- 常见技术点：工具装饰器模式、checkpointer 机制、流式输出、编码处理、字体配置

### 七、依赖说明

- 表格：库名 / 用途
- 仅列 `< 10` 个核心依赖
- 标准库（os、sys、pathlib）不用列
- 间接依赖不用列

### 八、运行方式

- **环境要求**：Python 版本、需要安装的额外服务（如 Ollama）
- **环境变量**：用代码块展示 `.env` 内容
- **运行命令**：可直接复制执行的完整命令
- **预期输出**：运行后应该看到什么

## 质量检查清单

- [ ] 文件结构图与实际目录一致
- [ ] 架构图正确反映组件调用关系（主代理→子代理→工具层级）
- [ ] 每个工具的输入/输出/后端/位置都有说明
- [ ] 执行流程覆盖完整路径（从 `if __name__` 到最终输出）
- [ ] 关键技术点独立提炼，非简单罗列代码行
- [ ] 依赖表与实际 import 一致，无遗漏无多余
- [ ] 运行方式的命令可复制粘贴直接执行
- [ ] 环境变量有完整示例（含注释）
- [ ] 中文表述清晰，代码注释/字符串保留原文
- [ ] ASCII 框图字符对齐，无错位
- [ ] 章节间有 `---` 分隔，排版一致
- [ ] 表格无空的表头单元格
